Form Inputs

Form inputs are fields for inputting data. Several attributes which must be set for correct data management.

<label for="one">First Example: </label>
<input type="text" id="one" name="this">
<label for="two">Second Example: </label>
<input type="email" id="two" name="this">
<input type="submit"> <input type="reset">

The label for must match the id of an input.
The information from the above form would result in "one":"Data-one", "two":"Data-two"

Inputs

Button

A push button with no default behaviour. Displays the value of the vale attribute, empty by default.

Checkbox

A check box allowing single vales to be selected/deselected.

Color

A control for allowing the user to input a color. Opens a color picker in supporting browsers.

Date

Allows the user to input a date consisting of year, month and day. Opens a date picker in supporting browsers.

A default date can be entered in the value attribute with the format yyyy-mm-dd

Datetime-Local

A control for entering a date and time, with no time zone. Opens a date picker in supporting browsers.

A default value is entered as yyyy-mm-ddThh:mm

Email

Text box with special validating parameters. Displays relevant keyboard in supporting browsers.

File

Allows the user to select a file from their local directory. The accept attribute defines which types of file the input will accept.

Hidden

A control that is not displayed but whose value is submitted with the form information.
A hidden input may be edited in developer tools - not suitable for security.

<input type="hidden"  alt="Submit via Pokeball">

Image

A graphical submit button. Display an image with the src attribute, alt attribute will display if the image is missing.
Height and width attributes can be applied.

<input type="image" src="../assets/css/images/pokeball.png" alt="Submit via Pokeball">
