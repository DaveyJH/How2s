Divs and Spans

Semantically Neutral - Structural Elements

Divs

A div is a block element. It will fill the width of its container
It is possible to change the size of a div with CSS. A div takes no semantic meaning, it does allow for properties and attributes to be applied to its children.

Block Elements

Block elements take the entire width of their container/parent by default. Browsers typically put a space above and below.
Their size can be manipulated.

Spans

Spans are inline elements and so:continue without a break
Spans can nest inside each other but must not contain block elements. They take the role of allowing property and attribute changes to their contained children.

Inline Elements

Inline elements take the size of their content. They do not create a line break before or after.
It is not possible to change the size of an inline element.
