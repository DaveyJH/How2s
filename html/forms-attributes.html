Form Attributes

Form attributes specify where to send the form data and how to send it.

  • Action
  • Method
  • Target

<form action="https://send-my-data-here.xyz" method="POST">

<input type="email" name="unique">
<input type="submit">

</form>

Action

The action attribute has a value of the URL of where the data is to be sent. If the action attribute is omitted, the action is set to the current page.

Method

The method attribute has a value of either GET or POST.

  • post is used to push data into the database. eg Updating a contact number
  • get is used to fetch exisiting data. eg Check a username is valid.

Form data sent via GET uses URL variables, the resulting URL contains the data sent and so is not secure. POST uses HTTP post transaction and so the data is not visible. https means the server is secure. See runnable examples and further notes here.

Target

The target attribute specifies where to display the respone received after submitting the form. The default is  _self.

  •  _blank The respone is displayed in a new window.
  •  _self The respone is displayed in the current frame.
  •  _parent The response is displayed in the parent frame.
  •  _top The response if displayed in the full body of the current window.
  •  name-of-iframe the response is displayed in the named iframe.
